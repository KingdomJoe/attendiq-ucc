# AttendIQ UCC — Desktop IDE Error Triage Skill

## PURPOSE
This skill gives you the definitive diagnosis and resolution playbook for IDE errors in the `desktop/` module of AttendIQ UCC, specifically the recurring "cannot be resolved" errors on `App`, `SessionManager`, and `FxUtils` imports.

---

## The Root Cause (Confirmed)

**Maven compile result: `BUILD SUCCESS` — Nothing to compile - all classes are up to date.**

The IDE errors are **false positives caused by a stale language server index**. The files exist and compile correctly:

| Class | Location | Package |
|-------|----------|---------|
| `App` | `desktop/src/main/java/com/ucc/attendance/desktop/App.java` | `com.ucc.attendance.desktop` |
| `SessionManager` | `desktop/src/main/java/com/ucc/attendance/desktop/SessionManager.java` | `com.ucc.attendance.desktop` |
| `FxUtils` | `desktop/src/main/java/com/ucc/attendance/desktop/util/FxUtils.java` | `com.ucc.attendance.desktop.util` |

---

## Why This Happens in JavaFX Maven Projects

JavaFX Maven projects are particularly prone to IDE cache staleness because:

1. **Non-modular but module-path dependency:** JavaFX JARs are loaded as named modules on the module path even without `module-info.java`. IDEs sometimes lose track of this.
2. **Multi-module Maven:** The desktop is a sub-module. IDE re-indexing after changes to sibling modules (`backend/`) can corrupt the desktop module's classpath model.
3. **Fat-jar with maven-shade-plugin:** The shade plugin packs dependencies into one JAR; the IDE doesn't always understand the final classpath correctly.

## Fixes (Ordered by Invasiveness)

### Fix 1: Force Maven Refresh (Try First)
```powershell
# In project root:
mvn compile -pl desktop
```
If this outputs `BUILD SUCCESS`, the code is correct. The issue is purely the IDE.

### Fix 2: Cursor/VSCode Java Extension Cache Clear
1. Open Command Palette (`Ctrl+Shift+P`)
2. Type: **"Java: Clean Java Language Server Workspace"**
3. Click **"Restart and delete"** when prompted
4. Wait ~30 seconds for re-indexing
5. Reopen the affected file

### Fix 3: Force Re-Import Maven Projects
1. Open Command Palette (`Ctrl+Shift+P`)
2. Type: **"Java: Reload Projects"** or **"Maven: Reload All Maven Projects"**
3. Wait for completion

### Fix 4: Delete Target Directory and Force Recompile
```powershell
# Wipes compiled classes, forces fresh compile
Remove-Item -Recurse -Force "C:\Users\PulPiT\.cursor\UCC-Attendance-system\desktop\target"
mvn compile -pl desktop
```
Then reload the Maven project in the IDE.

### Fix 5: Nuclear Option — Delete `.classpath` / `.project` / `.factorypath`
```powershell
# Only if all else fails — these are auto-regenerated by Maven
$desktopDir = "C:\Users\PulPiT\.cursor\UCC-Attendance-system\desktop"
Remove-Item "$desktopDir\.classpath" -ErrorAction SilentlyContinue
Remove-Item "$desktopDir\.project" -ErrorAction SilentlyContinue
Remove-Item "$desktopDir\.factorypath" -ErrorAction SilentlyContinue
mvn eclipse:eclipse -pl desktop  # Regenerate Eclipse project files (if using Eclipse)
```

## Other Warnings (Non-Critical)

### ApiClient.java line 183: `getErrorMessage(String)` never used locally
This is also a false positive. The overloaded method `getErrorMessage(String responseBody)` IS used — it delegates to `getErrorMessage(String, int)`. This is an internal private helper. The warning is benign.

### WebController.java line 38: `@SuppressWarnings("null")` info
The `@SuppressWarnings("null")` annotation suppresses null-safety analysis warnings from the IDE's null checker. This is intentional — it prevents false "potential null dereference" warnings for Lombok-generated null-safe code.

## How to Know if an Error is REAL vs. IDE Cache

Run Maven compile from the terminal:
```powershell
cd C:\Users\PulPiT\.cursor\UCC-Attendance-system
mvn compile -pl desktop 2>&1
```

| Result | Interpretation |
|--------|---------------|
| `BUILD SUCCESS` | All errors are IDE cache issues — ignore them |
| `BUILD FAILURE` with `error: cannot find symbol` | Real compile error — needs fixing |
| `BUILD FAILURE` with `FlywayException` | Database migration issue (backend, not desktop) |
